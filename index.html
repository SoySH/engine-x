Panel web para la gestión avanzada de sitios NGINX.

Administra de forma centralizada tus archivos *.conf de NGINX vanilla mediante un panel web seguro, intuitivo y ligero.
Descarga el script, configura tu base de datos para el acceso, conéctate por IP interna usando TLS autogenerado (certificado autofirmado) y gestiona tus sitios directamente desde tu propio servidor.

Página de inicio

Accede rápidamente al panel y visualiza tu token dinámico por sesión para mayor seguridad.

Panel de creación

Crea nuevos sitios con un solo clic utilizando la plantilla predefinida.

Edición en caliente

Edita configuraciones en producción con validación de sintaxis en tiempo real.
El sistema genera backups automáticos y restaura el último estado funcional si se detectan errores, evitando caídas del servidor.
Atajo de guardado: Ctrl + S

Botones dedicados

Edita, habilita, deshabilita, renombra o elimina sitios fácilmente, sin necesidad de reiniciar NGINX.

Requisitos

Sistema operativo: Ubuntu 22.04 o superior.

Conectividad a Internet (necesaria únicamente para descargas iniciales).

NGINX instalado vía gestor de paquetes (o compilado manualmente: requiere definir rutas personalizadas en api.php).

MariaDB
(almacena usuarios, contraseñas cifradas y tokens de sesión autodestruibles al cerrar sesión).

PHP (puede usar el existente o instalarse desde repositorio).

OpenSSL (generación de certificado TLS autofirmado).

Casos de uso ideales

Homelab y servidores personales.

Entornos de desarrollo y staging.

Administración directa de VirtualHosts NGINX.

Proyectos autoalojados con enfoque en control y simplicidad.

Limitaciones conocidas

  • No está diseñado para entornos multiusuario masivos.
  • No sustituye herramientas de orquestación (Ansible, Puppet, etc.).
  • La seguridad está orientada a redes internas, no exposición pública directa.
  • No gestiona certificados TLS de autoridades externas (Let’s Encrypt, ACME, etc.).
  • No valida lógica de negocio dentro de los archivos .conf, únicamente sintaxis de NGINX.
  • No incluye control de versiones histórico (solo backup inmediato).

Sobre seguridad

AVISO IMPORTANTE:

Este sistema requiere una entrada en sudoers ubicada en: /var/www/sudoers.d/enginex-www-data
Se recomienda su uso exclusivamente en entornos internos.

Medidas implementadas:

* Token dinámico con autoexpiración tras 2 horas.

* Bloqueo por mal uso: 5 intentos fallidos por usuario. Baneo automático de 2 horas.

* Uso de TLS provisional mediante certificado autofirmado.

* Permisos explícitos al usuario www-data: Archivos: 640, Directorios: 755, Sudoers: 440 (enginex)

FAQ

¿Por qué comenzar en la versión 3.5?

Las versiones 1 a 3 fueron iteraciones internas de desarrollo.
Se libera directamente v3.5 como primera versión pública estable.

¿Es gratuito?

Sí. Engine-X Sites es un proyecto de uso libre.

¿Requiere acceso root?

No directamente.
El panel opera mediante permisos específicos configurados en sudoers para el usuario www-data.

¿Puedo usarlo en producción?

Está orientado principalmente a entornos internos, homelab, desarrollo y staging.
Su uso en producción queda bajo responsabilidad del administrador.

¿Gestiona certificados de Let’s Encrypt u otros ACME?

No.
Actualmente solo genera certificados TLS autofirmados en la instalación para habilitar HTTPS.

¿Soporta múltiples usuarios?

El sistema incluye base para manejo de usuarios.
Por defecto se instala con un usuario inicial.

¿Reinicia NGINX al realizar cambios?

Los archivos de respaldo se guardan localmente en el servidor junto a la estructura del panel.

¿Tiene actualizaciones automáticas?

Las actualizaciones se aplican manualmente.

Acerca de
Engine-X Sites es un proyecto desarrollado como herramienta personal para simplificar la administración de sitios NGINX en entornos autoalojados.
Con el tiempo evolucionó hasta convertirse en un panel funcional orientado a administradores que prefieren control directo, simplicidad y operación local.
